4 Princípios de Bancos de dados

Durabilidade

Todo dado que é colocado,alterado ou manipulado tem que permanecer durável enquanto você quiser que ele esteja lá.

Isolamento

Se você tiver duas transações feitas ao mesmo tempo elas tem que ser executadas sem interferir uma a outra.

Consistência

Toda transação tem que levar o banco de dados de um estado consistente a outro consistente

Atomidade

Toda transação tem que ser atômica, ou toda ela acontece ou nada acontece.

Iniciar MySql server

Em linux debian/ubuntu:

Iniciar servidor: /etc/init.d/mysql start

Parar: /etc/init.d/mysql stop

Reiniciar: /etc/init.d/mysql restart

Acessando MySql via terminal

mysql -h localhost -u root -p

mysql => chama o servidor

-h => informamos que o proximo dado é referente ao host

localhost => informamaos o host

-u => infromamos que o proximo dado é referente ao usuario

root => informamos o usuario

-p => informamos que o proximo dado é a senha

Criando Banco de Dados

Para criar um banco de dados Basta usar o seguinte comando:

CREATE DATABASE nome_do_banco_de_dados

lembrando sempre de dar o USE nome_do_banco_de_dados

Criando Tabelas

CREATE TABLE nome_da_tabela (

id INT AUTO_INCREMENT NOT NULL,

campo1 INT,

campo2 VARCHAR (255)

PRIMARY KEY (id)

)

Primary Key é uma constraint que identifica exclusivamente cada registro em uma tabela. deve conter valores UNIQUE e não pode conter valores NULL. Uma tabela só pode ter UMA Primary key, e na tabela,essa chave primária pode consistir em uma ou varias colunas/campos.

INT declara que o campo1 só recebera valores inteiros.

VARCHAR é um conjunto de dados de caracteres de comprimento indeterminado, que pode conter letras e números. Diferente de CHAR que é um comprimento fixo, o varchar só cria posições que serão preenchidas, reduzindo assim o espaço.

Exemplo:

Se você tiver um CHAR(10) que armazene um valor de apenas 3 posições, as outras 7 posições serão criadas e não serão preenchidas.

However se você criar um VARCHAR(10) que armazene também um valor de 3 posições só serão criada essas 3 posições.

OBS: CHAR é mais usado para valores com comprimento fixo como sigla de estado,CEP etc...

AUTO INCREMENT permite que um número único seja gerado quando um novo registro é inserido em uma tabela, em mysql a Primary Key, cujo valor inicial é 1 e se incrementa de 1 em 1.

NOT NULL essa constraint garante que uma coluna nao admita valores NULL.

Estrutura da Tabela

Para ver que comando você usou para criar determinada tabela use:

show create table sua_tabela

Para ver os tipos de dados e mais informações da tabela use:

describe nome_da_tabela

Alterando Tabelas

Inserindo dados...

SINTAX:

insert into nome_tabela

(coluna1,coluna2,coluna3)

values

(valor1,valor2,valor3)

;

Exemplo:

insert into contatos

(nome,ddd,telefone)

values

('Ebert', '75', '99243-4896');

Alterando Estrutura...

Adicionando coluna com add column:

alter table contatos

add column profissao varchar (10);

Removendo coluna com drop column

alter table contatos

drop column profissao

Se você quiser colocar uma coluna como primeira use o first no fim do comando para criar ela. Se você quiser posicionar depois de alguma coluna use o after nome_da_coluna, caso queira a coluna como ultima basta não usar nenhum dos dois.
