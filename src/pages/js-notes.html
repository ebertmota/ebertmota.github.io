Arrow Function

As Arrow Functions foram introduzidas no ES6.

e nos permite escrever uma função com uma sintax mais curta.

Exemplo de função sem sintex Arrow:

hello = function() {

return "Hello World";

};

Exemplo com Arrow Function:

hello = () => {

return "Hello World";

};

Se a função tiver e receber somente uma instrução ela pode ficar ainda mais curta:

hello = () => "Hello World";

Arrow Function com parâmetros

hello = (val) => "Hello World" + val ;

Se voce tiver somente um parâmetro pode tirar os parênteses:

hello = val => "Hello World" + val ;

Configurando projeto com Babel e Yarn

Apos ter instalado Node Js e o Yarn....

Dentro da pasta de seu projeto rode yarn init

Neste momento o comando cria um arquivo package.json dentro de seu projeto, este arquivo vai armazenar as informações de depencias da nossa aplicação como o documento abaixo:

{

"name" : "es6",

"version" : "1.0.0",

"main" : "index.js",

"license" : "MIT",

}

Instalando Dependencias do Babel

@babel/cli

// Permite trabalhar com a interface gráfica do babel em linha de comando

@babel/preset-env

// Vai entender qual é o ambiente que você está trabalhando (Ex: browser,Node,React etc)

@babel/core

// Esse modulo core é o centro nervoso do babel

As dependencias irão criar um arquivo yarn.lock onde será armazenado o cache do Yarn, e uma pasta node_modules que irá armazenar todas as dependencias do projeto(as dependencias das dependencias etc) OBS: adicionar a pasta node_modules/ no seu .gitignore

Configurando Babel

Para fazer a configuração crie um arquivo na raiz de seu projeto com o nome de .babelrc, e dentro desse arquivo adicione as seguintes linhas:

{

"presets" : ["@babel/preset-env"]

}

Agora dentro do seu packge.json você criará uma nova propriedade:

"scripts" : {

"dev" : "babel ./main.js -o ./bundle.js"

}

o "scripts" é onde você pode adicionar funcionalidades para usar no terminal com o yarn

o "dev" é o nome do seu script que executa o babel

"babel ./main.js -o bundle.js -w" => basicamente executa o babel passando os parametros -o e -w.

O -o envia o conteudo do main.js para o bundle.js tornando ele compativel

E o -w ativa a monitoração para ele atualizar as alterações no main.js automaticamente sem ser necessario você rodar o comando toda hora.

Assim então toda funcionalidade nova do Javascript que você inserir no main.js sera convertida para ser compativel em diversos browsers que não suportam ainda estas funcionalidades, e toda essa conversão vai pro arquivo bundle.js

Por isso chame o bundle.js em vez do main.js na sua pagina html.

Depois de todo o procedimento nosso package.json ficará assim:

{

"name" : "es6",

"version" : "1.0.0",

"main" : "index.js",

"license" : "MIT",

"dependencies" : {

"@babel/cli" : "^7.6.2",

Class

Uma classe é um tipo de função, mas em vez de usar a keyword function para inciá-la, usamos a keyword class, e as propriedades são atribuídas dentro de um método constructor().

constructor

O Constructor é um método especial para criar e inicializar um objeto criado a partir de uma classe.

Apenas um método especial com o nome constructor pode existir em uma classe, se a classe conter mais de um constructor() será mostrado o erro SyntaxError.

É o primeiro método instanciado todo vez que executarmos uma classe.

No constructor podemos fazer ações que disparam assim que o objeto é criado ou iniciar variaveis.

Definição

Use a keyword class para criar uma classe e sempre adicione um metodo constructor.

O método constructor vai ser chamado cada vez que o objeto for iniciado.

Exemplo:

class Carro {

constructor (marca, ano, situacao){

this.nome_do_carro = marca;

this.ano_do_carro = ano;

this.situacao_do_carro = situacao;

};

};

let meuGolQuadrado = new Carro ('Gol Quadrado', '2010', 'Usado');

console.log(meuGolQuadrado.nome_do_carro +meuGolQuadrado.ano_do_carro+meuGolQuadrado.situacao_do_carro)

Console > Gol Quadrado 2010 usado

Metodos

O método contructor é especial, é onde você inicializa propriedade, é chamado automaticamente quando uma classe é iniciada.

E ele precisa ter o nome exato "constructor", de fato se você não tiver um método constructor, o Javascript adicionará um método constructor invisível e vazio.

Você também é livre para criar seus próprios métodos, a sintaxe deve ser familiar:

class Carro {

constructor (marca){

this.nome_do_carro = marca;

}

present() {

return "Eu tenho um " + this.nome_do_carro;

}

}

var meuCarro = new Carro('Gol Quadrado');

console.log( meuCarro.present());

Como você viu no exemplo acima, você chama o método present() que consulta o método contructor, você pode passar qualquer parâmetro entre parênteses para o método present('qualquerParametro')

Métodos Static

Os métodos Static são definidos na própria classe e não no prototype.

Isso significa que você não pode chamar um método Static no objeto (meuCarro), mas na classe (Carro)

class Carro {

constructor (marca){

this.nome_do_carro = marca;

}

static Hello(parametro){

return "Hello " + parametro.nome_do_carro;

}

}

var meuCarro = new Carro('Gol Quadrado');

console.log( Carro.Hello(meuCarro));

Sub Classes ou Herança/Inheritance

Para criar uma classe com herança, use a keyword extends.

A classe criada com extends herda todos os métodos da outra classe.

Exemplo:

class Carro {

constructor (marca){

this.nome_do_carro = marca;

}

present() {

return "Eu tenho um " + this.nome_do_carro;

}

}

class Modelo extends Carro {

constructor (marca,modelo) {

super(marca);

this.modelo = modelo;

}

show() {

return this.present() + ', que é um ' + this.modelo;

}

}

var meuCarro = new Modelo ('Ford', 'Mustang');

console.log ( meuCarro.show() );

O método super() se refere a classe do pai.

Ao chamar o método super() no método constructor, chamamos o método constructor do Pai e obtemos acesso às propriedades e métodos do pai.

Promisses

Promisses são um conceito essencial do JavaScript. Elas estão presentes em praticamente todo o ecossistema da linguagem.

Promisses são um padrão de desenvolvimento que visam representar a conclusão de operações assincronas.

Exemplo de uma função normal:

function sumNumbers ( ) {

let result 1 + 1;

if (result == 2) {

sucessCallback();

} else {

errorCallback( ) ;

}

}

function sucessCallback( ) {

console.log (' YEAH BITCH! Number 2 ')

}

function errorCallback ( ) {

console.log (' Something is wrong... ');

}

Exemplo dessa mesma função com Promise:

//com promisse

//uma promisse sempre recebe uma função com dois parâmetros:

//resolve e reject

let p = new Promise ( ( resolve, reject ) => {

let a = 1 + 1 ;

if ( a == 2 ) {

resolve ( 'Sucess' )

} else {

reject ( 'Failed' )

}

} ) ;

//chamando promisse

//then para RESOLVE

//catch para REJECT

p.then( ( message ) => {

console.log ( 'This is in the then ' + message);

}); .catch ( ( error ) => {

console.log ( 'This is in the cath' + error);

});

Syntax

Utilizamos o prefixo async antes de definir uma função para indicar que estamos tratando de um código assíncrono, e com o prefixo adicionado, podemos utilizar o await antes das Promises indicando um ponto a ser aguardado pelo código. Exemplo:

// Modelo Antigo ( ES6 )

function fetchUser ( user ) {

api.get (`/users/${user }`)}).then (response => {

console.log ( response );

});

}

// Modelo Novo ( ES8 )

async function fetchUser ( user ) {

const response = await api.get (`/users/${user});

console.log( response );

}

Ao utilizar o async/await o código fica muito mais limpo, e você não precisa declarar mais os .then e .catch

Cada vez que definimos um await estamos indicando para o nosso código aguardar a Promise seguinte executar e retornar um resultado para seguir adiante, dessa forma evitamos que as próximas linhas executem sem as variáveis necessárias.

Vale lembrar que toda função que definimos com async automaticamente se torna uma Promise, tornando possíveis anexar funções assíncronas uma às outras.

Importação/exportação

A importação e exportação de módulos permite a utilização de funções, variáveis e classes de um arquivo em outro, vamos entender como isso funciona:

// App.js

export function soma(a, b) {

return a + b;

}

export const user = {

nome: 'Ebert' ,

empresa: 'RocketSeat

};

export default class App {

static log)() {

console.log( 'Hey' );

}

}

Nesse primeiro arquivo estamos exportando uma função soma, um objeto user e uma classe App.

Note que na última exportação estamos passando a palavra default e dessa forma estamos sinalizando que se não informamos a esse arquivo o quê estamos importando, por padrão ele deve retornar essa classe.

O export default só pode ser utilizado uma vez em cada arquivo Javascript

Vamos ver as importações agora:

// OutroArquivo.js

import App from 'App' ; // Podemos omitir o .js aqui

App.log( ); // Hey

soma( 1, 2) // Error: undefined function soma

Note que ele importou apenas a class App e nada mais. Para importarmos variáveis, objetos ou classes que não estão com a notação default podemos utilizar a Desestruturação.

Exemplo:

// OutroArquivo.js

import { soma, user } from 'App' ;

soma( 1, 3); // 4

console.log (user.nome) // Ebert

Outra forma de importar todas as exportações sem a notação default de um arquivo, é utilizando o wildcard *;

import * as helpers from 'App' ;

helpers.soma (1, 3) // 4

console.log (helpers.user.nome); // Ebert

Ainda podemos unir a importação do default e das outras exportações em um único import.

import App, { soma, user } from 'App' ;
