Factory Functions

Em Javascript qualquer função pode retornar um Objeto.

Quando uma função retorna um Objeto sem o uso da keyword new é uma Factory Function. Exemplo:

function personFactory (name) {

return {

name: name ,

type: 'person'

}

}

console.log( personFactory( 'Ebert' ));

Vantagens Desvantagens
Facilita a criação de objetos com o reuso Um pouco mais burocrática e verbosa em relação a constructor
Mais simples e mais clara que a constructor Um pouco menos performática em relação a constructor
O possível uso de Closures.
Restringe o uso do this.




Constructor Funtions

Por convenção sempre criar Constructor Functions usando a primeira letra em Uppercase.

Um constructor é basicamente uma função. Ela pode ser executada como uma função ou pode ser utilizada para instanciar um objeto utilizando a keyword new

Se você não usar o new, a função não utiliza o prototype e é executada como uma função qualquer. E o this é referenciado no escopo global -window

Exemplo:

function Cachorro( nome , idade ) {

this.nome = nome

this.idade = idade

}

const cachorro = new Cachorro ('Ebert', 18) ;

Vantagens Desvantagens
Menos verbosa Sempre que instanciarmos um novo objeto, será criado um espaço na memória.
Possibilidade de criar métodos e propriedades privadas Utiliza o operador new que pode ser incompreendido por muitos.
Reusabilidade Uso do this.
Ideais para objetos que podem existir como múltiplas instâncias no mesmo contexto.
prototype é mais eficiente tanto em uso de memória quanto de CPU.
Instanciar um objeto sem se preocupar com efeitos colaterais.
Ajuda a ter um código padronizado, flexível e seguro.

IIFE => Imediately invoked function expression

Uma IIFE é uma função que se auto-executa, e gera um novo escopo dentro dela própria.

(

function () {

console.log( Não estou tocando no escopo global )

}

) ( );

No exemplo acima, envolvemos a declaração da função em parênteses e acrescentamos mais um par de parênteses no final, invocando assim a função de imediato.

Uma IIFE pode ser criada com nome ou anônima como no exemplo acima.

Generator function

A declaração de function* (seguida de um asterisco) define uma função geradora que retorna um object Generator

Funções geradoras são funções que podem ter sua execução "pausada" e posteriormente resumida, seus contextos de associações de variavel ficarão salvos entre cada execução.

function* geradora1() {

yield 'Valor 1' ;

yield 'Valor 2' ;

yield 'Valor 3' ;

}

Chamar um função geradora não executa seu conteúdo imediatamente, ao invés disso um objeto iterator é retornado. Quando o objeto next() é chamado, o conteudo da função geradora é executado até a primeira expressão yield.

O método next() retorna um objeto com uma propriedade value contendo o valor retornado, e a propriedade boolean done indicando se o gerador produziu seu ultimo valor.

Uma expressão return em um gerador quando executada fará com que o gerador termine ou seja a propriedade done do objeto retornado terá o valor true.

Exemplo útil como Generator functions:

function* idGenerator() {

let index = 0 ;

while ( true ) {

yield index ;

index++ ;

}

}

const id = idGenerator() ;

console.log( id.next( ).value ); // Toda vez q a função for chamada um numero de 0 ao infinito será retornado

Arrow Function

As Arrow Functions foram introduzidas no ES6.

e nos permite escrever uma função com uma sintax mais curta.

Exemplo de função sem sintex Arrow:

hello = function() {

};

Exemplo com Arrow Function:

hello = () => {

};

Se a função tiver e receber somente uma instrução ela pode ficar ainda mais curta:

hello = () => "Hello World";

Arrow Function com parâmetros

Filter

O método Filter cria um novo array com todos os elementos que passaram no teste implementado pela função fornecida.

Sintax

const newArray = array.filter( callback);

// Callback é uma função que testa cada elemento do array, retornando true para manter o elemento, e false para remover, o Callback recebe 3 argumentos.

Element => o elemento que está sendo processado no array, ou seja o valor de cada item

Index => O indice do elemento atual que está sendo processado no array

array => O proprio array que é retornado a cada iteração no filter

Veja um exemplo simples:

const numeros = [ 5, 50, 80, 1, 2, 3, 5, 8, 7, 11, 15, 22, 27 ];

// Retornar todos os números maiores que 10

const numerosFiltrados = numeros.filter ((value, index, array) => {

if (value > 10){

return true;

}

else {

return false;

}

});

// [ 50, 80, 11, 15, 22, 27 ]

const numerosFiltrados = numeros.filter (value => value > 10);

Reduce

Como o nome sugere, o reduce busca reduzir um array. Ele iterará por cada elemento dessa lista com o objetivo de ao final gerar um único valor (de qualquer tipo), como por exemplo a soma de todos os elementos desse array. Lembrando que não ficamos presos apenas a números.

Sintax:

array.reduce (function (acumulator, currentValue, index, array) {} , acumulatorInitialValue);

// Acumulator é a variável que vai servir como acumuladora (que é o valor retornado na ultima iteração) (obrigatório).

// acumulatorInitialValue será o valor que a variável acumuladora iniciará, Se não passarmos nada, o valor do acumulator será o primeiro valor do array, e o valor atual(2 parâmetro do callback value) iniciará pelo segundo.

Exemplos:

const numeros = [5, 50, 80, 1, 2, 3, 5, 8, 7, 11, 15, 22, 27];

// Retorne a soma de todos os numeros

const soma = numeros.reduce ( (acumulator, value) => {

acumulator += value; return acumulator;

}, 0 );

// 236

O reduce também pode cumprir o papel do map e do filter o que não é recomendado mas que veremos em exemplos abaixo:

//Retorne um array com os numeros pares (Filter)

const pares = numeros.reduce ( (acumulator, value) => {

if (value % 2 === 0) acumulator.push(value);

return acumulator;

}, [ ]);

// [ 50, 80, 2, 8, 22 ]

//Retorne um Array com o Dobro dos valores (MAP)

const dobro = numeros.reduce ( (acumulator, value) => {

acumulator.push (value * 2);

return acumulator;

}, [ ]);

// [ 10, 100, 160, 2, 4, 6, 10, 16, 14, 22, 30, 44, 54 ]

For Each

O método forEach() chama uma função uma vez para cada elemento em um array, em ordem.

Sintax

array.forEach (function (currentValue, index, array), thisValue)

Exemplos:

// Retorne a soma de todos os valores em um Array

let sum = 0;

const numbers = [ 60, 9, 24, 12 ];

numbers.forEach( ( item ) => {

});

console.log(sum);

// 105

// Multiplique o valor de cada elemento por 10

const numbers = [ 60, 9, 24, 12 ];

numbers.forEach(myFunction);

function myFunction( item, index, array) {

array[index] = item * 10

}

// [ 600, 90, 240, 120 ]

O -o envia o conteudo do main.js para o bundle.js tornando ele compativel

E o -w ativa a monitoração para ele atualizar as alterações no main.js automaticamente sem ser necessario você rodar o comando toda hora.

Assim então toda funcionalidade nova do Javascript que você inserir no main.js sera convertida para ser compativel em diversos browsers que não suportam ainda estas funcionalidades, e toda essa conversão vai pro arquivo bundle.js

Por isso chame o bundle.js em vez do main.js na sua pagina html.

Class

Uma classe é um tipo de função, mas em vez de usar a keyword function para inciá-la, usamos a keyword class, e as propriedades são atribuídas dentro de um método constructor().

constructor

O Constructor é um método especial para criar e inicializar um objeto criado a partir de uma classe.

Apenas um método especial com o nome constructor pode existir em uma classe, se a classe conter mais de um constructor() será mostrado o erro SyntaxError.

É o primeiro método instanciado todo vez que executarmos uma classe.

No constructor podemos fazer ações que disparam assim que o objeto é criado ou iniciar variaveis.

Definição

Use a keyword class para criar uma classe e sempre adicione um metodo constructor.

O método constructor vai ser chamado cada vez que o objeto for iniciado.

Exemplo:

class Carro {

constructor (marca, ano, situacao){

this.nome_do_carro = marca;

this.ano_do_carro = ano;

this.situacao_do_carro = situacao;

};

};

let meuGolQuadrado = new Carro ('Gol Quadrado', '2010', 'Usado');

console.log(meuGolQuadrado.nome_do_carro +meuGolQuadrado.ano_do_carro+meuGolQuadrado.situacao_do_carro)

Console > Gol Quadrado 2010 usado

Metodos

O método contructor é especial, é onde você inicializa propriedade, é chamado automaticamente quando uma classe é iniciada.

E ele precisa ter o nome exato "constructor", de fato se você não tiver um método constructor, o Javascript adicionará um método constructor invisível e vazio.

Você também é livre para criar seus próprios métodos, a sintaxe deve ser familiar:

class Carro {

constructor (marca){

this.nome_do_carro = marca;

}

present() {

return "Eu tenho um " + this.nome_do_carro;

}

}

var meuCarro = new Carro('Gol Quadrado');

console.log( meuCarro.present());

Como você viu no exemplo acima, você chama o método present() que consulta o método contructor, você pode passar qualquer parâmetro entre parênteses para o método present('qualquerParametro')

Métodos Static

Os métodos Static são definidos na própria classe e não no prototype.

Isso significa que você não pode chamar um método Static no objeto (meuCarro), mas na classe (Carro)

class Carro {

constructor (marca){

this.nome_do_carro = marca;

}

static Hello(parametro){

return "Hello " + parametro.nome_do_carro;

}

}

var meuCarro = new Carro('Gol Quadrado');

console.log( Carro.Hello(meuCarro));

Promises

Promises são um conceito essencial do JavaScript. Elas estão presentes em praticamente todo o ecossistema da linguagem.

Promises são um padrão de desenvolvimento que visam representar a conclusão de operações assincronas.

Exemplo de uma função normal:

function sumNumbers ( ) {

let result 1 + 1;

if (result == 2) {

sucessCallback();

} else {

errorCallback( ) ;

}

}

function sucessCallback( ) {

console.log (' YEAH BITCH! Number 2 ')

}

function errorCallback ( ) {

console.log (' Something is wrong... ');

}

Exemplo dessa mesma função com Promise:

//com promisse

//uma promisse sempre recebe uma função com dois parâmetros:

//resolve e reject

let p = new Promise ( ( resolve, reject ) => {

let a = 1 + 1 ;

if ( a == 2 ) {

resolve ( 'Sucess' )

} else {

reject ( 'Failed' )

}

} ) ;

//chamando promisse

//then para RESOLVE

//catch para REJECT

p.then( ( message ) => {

console.log ( 'This is in the then ' + message);

}); .catch ( ( error ) => {

console.log ( 'This is in the cath' + error);

});

Syntax

Utilizamos o prefixo async antes de definir uma função para indicar que estamos tratando de um código assíncrono, e com o prefixo adicionado, podemos utilizar o await antes das Promises indicando um ponto a ser aguardado pelo código. Exemplo:

// Modelo Antigo ( ES6 )

function fetchUser ( user ) {

api.get (`/users/${user }`)}).then (response => {

console.log ( response );

});

}

// Modelo Novo ( ES8 )

async function fetchUser ( user ) {

const response = await api.get (`/users/${user});

console.log( response );

}

Ao utilizar o async/await o código fica muito mais limpo, e você não precisa declarar mais os .then e .catch

Cada vez que definimos um await estamos indicando para o nosso código aguardar a Promise seguinte executar e retornar um resultado para seguir adiante, dessa forma evitamos que as próximas linhas executem sem as variáveis necessárias.

Vale lembrar que toda função que definimos com async automaticamente se torna uma Promise, tornando possíveis anexar funções assíncronas uma às outras.

Importação/exportação

A importação e exportação de módulos permite a utilização de funções, variáveis e classes de um arquivo em outro, vamos entender como isso funciona:

// App.js

export function soma(a, b) {

return a + b;

}

export const user = {

nome: 'Ebert' ,

empresa: 'RocketSeat

};

export default class App {

static log)() {

console.log( 'Hey' );

}

}

Nesse primeiro arquivo estamos exportando uma função soma, um objeto user e uma classe App.

Note que na última exportação estamos passando a palavra default e dessa forma estamos sinalizando que se não informamos a esse arquivo o quê estamos importando, por padrão ele deve retornar essa classe.

O export default só pode ser utilizado uma vez em cada arquivo Javascript

Vamos ver as importações agora:

// OutroArquivo.js

import App from 'App' ; // Podemos omitir o .js aqui

App.log( ); // Hey

soma( 1, 2) // Error: undefined function soma

Note que ele importou apenas a class App e nada mais. Para importarmos variáveis, objetos ou classes que não estão com a notação default podemos utilizar a Desestruturação.

Exemplo:

// OutroArquivo.js

import { soma, user } from 'App' ;

soma( 1, 3); // 4

console.log (user.nome) // Ebert

Outra forma de importar todas as exportações sem a notação default de um arquivo, é utilizando o wildcard *;

import * as helpers from 'App' ;

helpers.soma (1, 3) // 4

console.log (helpers.user.nome); // Ebert

Ainda podemos unir a importação do default e das outras exportações em um único import.

import App, { soma, user } from 'App' ;

Módulos

O que são módulos?

Como o capítulo de um livro, os módulos são apenas conjunto de palavras (ou códigos).

Bons módulos, no entanto, são altamente auto-suficiente com funcionalidades distintas,o que lhes permite ser misturados, removidos ou adicionaos conforme a necessidade sem impactar no sistema como um todo.

Porquê usar ?

Existem uma série de benefícios ao usar módulos em favor de uma extensa base de códigos interdenpendentes. Os mais importantes geralmente são:

1) Manutenibilidade: Por definição, um módulo é auto-suficiente, quando bem projetado visa diminuir as dependências em partes da base de código, tanto quanto possível, de modo que ele possa crescer e melhorar de forma independentes. Atualizando um único módulo é muito mais fácil quando o mesmo é dissociado de outros pedaços de códigos.

Voltando ao exemplo do livro, se você quisesse atualizar um capítulo de seu livro, seria um pesadelo se para uma pequena alteração em um capítulo, você tivesse que ajustar todos os outros. Em vez disso voçê gostaria de escrever cada capítulo de tal forma que as melhorias poderiam ser feitas individualmente sem afetar uns aos outros.

2) Namespacing: Em JavaScript, variáveis fora do escopo de uma função de nível superior são globais. Devido a isso é comum ter a "poluição de namespace", onde completamente existem códigos compartilhados sem referência, variáveis globais.

Como veremos no final desse post , módulos nos permitem evitar poluição de namespace, criando um espaço privado para nossas variáveis.

Compartilhamento de variáveis globais entre códigos independentes é inaceitável em desenvolvimento

3) Reusabilidade: Vamos ser honestos, todos nós já copiamos códigos e escrevemos em novos projetos em um ponto ou outro. Por exemplo, vamos imaginar que você copiou alguns métodos utilitários que escreveu em um projeto anterior no seu projeto atual.

Isso tudo é bom, mas se você encontrar uma melhor maneira de escrever alguma parte desse código, voc}e teria que voltar e lembrar de atualizar em qualquer outro lugar que você escreveu.

Isso é obviamente um enorme desperdício de tempo. Não seria muito mais fácil se houvesse, por exemplo, um módulo que pudesse ser reutilizável?

Webpack

O que é webpack?

O webpack é um empacotador de módulos estáticos para aplicações javascript modernas. Ao processar a aplicação o webpack gera um grafico que mapeia cada módulo e suas dependências e gera um ou mais pacotes.

Conceitos Principais

Entry

Ponto de entrada que indica qual módulo o webpack deve usar para iniciar a construção do gráfico interno de dependência. Ao definir um ponto de entrada, o webpack irá encontrar todas as dependências e fazer a importação.

Por padrão o ponto de entrada é definido no arquivo ./src/index.js, mas é possível definir um arquivo diferente ou até mesmo múltiplos pontos de entrada no arquivo de configuração webpack.config.js exemplo de configuração do entry:

module.exports = {

entry: './path/to/my/entry/file.js'

}

Output

A propriedade Output define o nome e local do pacote gerado pelo webpack. O diretório padrão é o ./dist e o arquivo ./dist/main.js. Para configurar, devemos definir um objeto output com as propriedades path e filename no arquivo de configuração do webpack:

module.exports = {

entry: './path/to/my/entry/file.js'

output: {

path: path.resolve( __dirname, 'dist'),

filename: 'webpack.bundle.js'

}

}

Essa configuração gera um diretorio dist na raiz do projeto e o arquivo webpack.bundle.js no diretorio dist.

A variavel do nodeJS __dirname contém o nome do diretorio onde se encontra o módulo atual.

Loaders

Por padrão o Webpack só entende arquivos JavaScript, para que ele possa entender outros tipos de arquivos devemos usar os Loaders que são módulos que podem ser instalados separadamente possibilitando que o webpack converta esses arquivos em módulos válidos e os adicione ao gráfico de dependência.

Os Loaders também são utilizados para converter JavaScript de uma versão para outra. Para incluirmos os Loaders criamos uma nova seção module no arquivo de configuração, nessa seção podemos definir uma ou mais rules:

cons path = require('path')

const config = {

output: {

filename: 'my-first-webpack-bundle.js'

},

modules: {

rules: [

{

test: /\.js$/,

exclude: /node_modules/,

use: {

loader: 'babel-loader'

}

}

]

}

}

Essa regra diz ao compilador que quando for encontrado um arquivo que termine com .JS, ignorando os arquivos .js da pasta node_modules o babel-loader deverá ser utilizado para gerar o bundle.

test : define quais arquivos serão filtrados

use : define qual módulo será usado

Mode

No atributo mode é definido o mode de execução do webpack como produção, desenvolvimento ou nenhum. De acordo com a opção definida o webpack ativa otimizações específicas. O ambiente padrão é produção:

module.exports = {

mode: 'production'

}
